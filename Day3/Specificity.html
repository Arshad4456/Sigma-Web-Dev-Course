Specificity

The color of this heading is determined by the CSS rules applied to it. The order of specificity is as follows:

  • Inline styles (not used here)
  • ID selectors (#green)
  • Class selectors (.class1, .class2, .class3)
  • Element selectors (h1)
In this case, the ID selector has higher specificity than the class selectors, so the text color is green.

If you want to override the ID selector, you can use the !important rule in the class2 style. This will make the text color yellow, regardless of the specificity order.

Cascade Algorithm

The cascade algorithm determines which CSS rules apply to an element when multiple rules match. The order of precedence is as follows:

  1. Inline styles
  2. IDs
  3. Classes
  4. Elements
  5. Universal selector (*)

cascade has four distinct stages

  1. Position and order of appearance:
  2. Specificity
  3. Origin
  4. Important rules

specificity calculation

The calculation give different number for different selectors

  • Inline styles: 1000
  • ID selectors: 100
  • Class selectors: 10
  • Element selectors: 1
  • Universal selector (*): 0
  • Attribute selectors: 10
  • Pseudo-classes: 10
  • Pseudo-elements: 1
  • Important rules: 10000
  • Media queries: 0
  • Keyframes: 0
  • Custom properties: 0
  • CSS variables: 0
  • CSS functions: 0

For example, if you have the following selectors:

  • h1: 1
  • .class1: 10
  • #green: 100
  • h1.class2: 11
  • h1.class3: 11
  • h1.class4: 11

The specificity of the combined selectors is calculated by adding the values together. For example, if you have a selector like h1.class1#green, the specificity would be 1 + 10 + 100 = 111.

In this case, the ID selector (#green) has the highest specificity, so it will take precedence over the class selectors and element selectors.

In conclusion, understanding specificity and the cascade algorithm is crucial for writing effective CSS. By knowing how to calculate specificity and the order of precedence, you can ensure that your styles are applied correctly and consistently across your web pages.
